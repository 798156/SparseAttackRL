## 📈 关键发现

### 发现1：准确率与鲁棒性正相关 ⭐⭐⭐⭐⭐

```
模型准确率排序：
VGG16 (92.27%) > MobileNetV2 (84.90%) > ResNet18 (83.77%)

鲁棒性排序（平均ASR，越低越好）：
VGG16 (60%) < ResNet18 (78%) < MobileNetV2 (93%)

结论：准确率越高 → 对抗鲁棒性越强
```

### 发现2：轻量级网络的鲁棒性代价 ⭐⭐⭐⭐⭐

```
MobileNetV2（轻量级设计）：
- 平均ASR: 93.3%（最脆弱）
- 平均L0: 5.16（最容易修改）

启示：
⚠️ 移动端/边缘设备部署需要考虑鲁棒性
⚠️ 效率-安全的权衡
```

### 发现3：JSMA vs SparseFool

```
JSMA整体略优：
- ResNet18:  JSMA 83% > SparseFool 73%
- VGG16:     JSMA 70% > SparseFool 50%
- MobileNetV2: 持平 93%

SparseFool更快：
- 平均时间：0.16秒 vs 0.46秒
- 快3倍
```

## 💡 经验教训

### 成功经验

1. **参数统一至关重要**
   - 前期因参数不统一浪费时间
   - 通过诊断找到合适参数
   
2. **先诊断再实验**
   - 10样本快速验证参数
   - 避免浪费时间跑错误配置

3. **质疑数据的科学态度**
   - 发现异常及时停止
   - 重新验证实现

4. **L0是关键指标**
   - Foolbox的L2攻击L0=1000+（不是稀疏）
   - 自己实现L0=3-6（真正稀疏）

### 失败教训

1. **One-Pixel攻击**
   - 太慢（3-5分钟/样本）
   - 参数敏感（popsize影响巨大）
   - 暂时放弃

2. **RL方法**
   - 训练困难
   - 效果不稳定
   - 不适合当前研究
